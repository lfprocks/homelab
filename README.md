# homelab

GitOps configuration for my homelab Kubernetes clusters, continuously
reconciled by [Flux](https://fluxcd.io/) **v2.3.0**. Flux watches this Git
repository and applies its contents to the cluster, so the desired state of
everything running lives here in git — git is the single source of truth.

## Clusters

| Cluster                                     | Status      | Notes                                                       |
| ------------------------------------------- | ----------- | ----------------------------------------------------------- |
| [`the-intersect`](clusters/the-intersect)   | **Active**  | The live cluster. Documented in detail below.               |
| `libvirt-unraid`                            | Deprecated  | Archived under [`clusters/_archive/`](clusters/_archive).   |
| `solar-pi`                                  | Deprecated  | Archived under [`clusters/_archive/`](clusters/_archive).   |

`libvirt-unraid` and `solar-pi` are no longer in use. Their entrypoints were
moved to [`clusters/_archive/`](clusters/_archive) to preserve their
configuration and history; nothing there is reconciled. See the
[archive README](clusters/_archive/README.md) for what was kept and how to
restore one.

## Repository layout

The repo follows the Flux
[monorepo structure](https://fluxcd.io/flux/guides/repository-structure/): each
cluster has an entrypoint under `clusters/`, which fans out to shared
`infrastructure/` and `apps/` directories. Common pieces live in `shared/` and
`base/` layers; each cluster opts in through its own overlay.

```
clusters/
  the-intersect/            # active cluster entrypoint (Flux Kustomizations)
  _archive/                 # deprecated clusters, not reconciled
infrastructure/
  controllers/
    shared/                 # controllers shared across clusters
    the-intersect/          # the-intersect controllers
  configs/
    the-intersect/          # the-intersect cluster config (issuers, gateways, ...)
apps/
  <app>/
    base/                   # shared base manifests for an app (where present)
    the-intersect/          # the-intersect overlay (kustomize over ../base)
scripts/                    # decrypt.sh / encrypt.sh / validate.sh helpers
```

## The `the-intersect` cluster

`the-intersect` is the only active cluster and the focus of this repo. It uses
[Cilium](https://cilium.io/) as the CNI and [Rook-Ceph](https://rook.io/) for
storage. Flux is bootstrapped against `clusters/the-intersect`, and everything
the cluster runs is described by the manifests reachable from that directory.

### How Flux is wired

`flux bootstrap` installs the Flux controllers and a root sync that points at
`clusters/the-intersect`. The root sync applies every YAML in that directory:

```
clusters/the-intersect/
  flux-system/
    gotk-components.yaml    # Flux controllers (generated by `flux bootstrap`)
    gotk-sync.yaml          # GitRepository + root sync Kustomization (generated)
    kustomization.yaml      # patches the root sync to enable SOPS decryption
  namespaces.yaml           # application namespaces (privileged PSA + gateway access)
  cluster-vars.sops.yaml    # SOPS-encrypted Secret of cluster-wide variables
  infrastructure.yaml       # shared-infra-controllers, infra-controllers, infra-configs
  apps.yaml                 # one Flux Kustomization per application
```

#### 1. Source and root sync — `flux-system`

`flux-system/gotk-sync.yaml` defines the `GitRepository` everything flows from.
It tracks the `main` branch of `https://github.com/lfprocks/homelab` and polls
every **1 minute**. The same file defines the root `Kustomization`
(`flux-system`), which reconciles `./clusters/the-intersect` every **10 minutes**
with `prune: true` — applying `namespaces.yaml`, `cluster-vars.sops.yaml`,
`infrastructure.yaml`, and `apps.yaml`.

`gotk-components.yaml` holds the Flux controllers (source, kustomize, helm, and
notification controllers). Both `gotk-*` files are generated by `flux bootstrap`
— **do not edit them by hand**.

`flux-system/kustomization.yaml` is the one hand-maintained piece in this
directory: it patches the root `Kustomization` to add `spec.decryption` (SOPS,
via the `sops-age` secret) so the root sync can decrypt and apply
`cluster-vars.sops.yaml`. It is applied as a patch so it survives `flux
bootstrap` regenerating `gotk-sync.yaml`.

#### 2. Infrastructure — `infrastructure.yaml`

Three `Kustomization` objects, chained with `dependsOn`:

- **`shared-infra-controllers`** → `./infrastructure/controllers/shared`
  (interval **1h**, `wait: true`, `prune: true`). Controllers common to every
  cluster.
- **`infra-controllers`** → `./infrastructure/controllers/the-intersect`
  (interval **1h**, `wait: true`, `prune: true`). Cluster-specific controllers.
- **`infra-configs`** → `./infrastructure/configs/the-intersect`, which
  **`dependsOn` both controller layers** so config (issuers, gateways, storage
  classes, etc.) is applied only after the controllers that consume it are ready
  (interval **1h**, `prune: true`).

#### 3. Apps — `apps.yaml`

Rather than a single apps Kustomization, each application has **its own**
`Kustomization` (`apps-<name>`) pointing at `apps/<app>/the-intersect`. Every
app **`dependsOn infra-configs`** (interval **10m**, `wait: true`, `prune: true`,
`timeout: 5m`), so workloads roll out only after their supporting infrastructure
is ready. A few have extra dependencies — e.g. `apps-adsbx-live-notifier` also
`dependsOn apps-pulsar-notification-pipeline`.

### Reconciliation order

```
GitRepository flux-system (poll every 1m)
        │
        ▼
root sync (./clusters/the-intersect)  ── applies namespaces + cluster-vars + the Kustomizations below
        │
        ├─ shared-infra-controllers ─┐
        ├─ infra-controllers ────────┴─► infra-configs ─► apps-* (per application)
        │      (1h)                          (1h)              (10m)
```

Every child Kustomization decrypts with SOPS (`sops-age`) and injects
cluster-wide values via `postBuild.substituteFrom` of the `cluster-vars` Secret.
Because `prune: true` is set throughout, deleting a manifest from git removes the
corresponding resource from the cluster on the next reconcile.

## Secret management

Secrets are committed encrypted with [SOPS](https://github.com/getsops/sops) +
[age](https://github.com/FiloSottile/age). `.sops.yaml` encrypts only the
`data`/`stringData` fields of YAML files (`encrypted_regex:
"^(data|stringData)$"`), using the age recipient in that file. Flux decrypts
them in-cluster with the `sops-age` secret.

### Prerequisites

- `sops` and `age` (`brew install sops age`)
- The repo's age private key at `age.agekey` (git-ignored — never committed)

### Common operations

```sh
# Decrypt a secret to stdout
SOPS_AGE_KEY_FILE=age.agekey sops -d apps/<app>/<cluster>/secret.yaml

# Encrypt a secret in place
sops -e --in-place apps/<app>/<cluster>/secret.yaml

# Push the age key into the cluster so Flux can decrypt (creates the sops-age
# secret in flux-system)
make age-secret-to-cluster
```

Helper scripts live in `scripts/` (`decrypt.sh`, `encrypt.sh`, `validate.sh`).

### Variable substitution (topology values)

Identifying values that aren't secret material but shouldn't be published in the
clear — hostnames, LAN IPs, NFS/SMB hosts, LoadBalancer/MetalLB ranges, the
ACME email, the managed-domain list — are kept out of manifests as `${TOKEN}`
placeholders and resolved at reconcile time via Flux
[post-build variable substitution](https://fluxcd.io/flux/components/kustomize/kustomizations/#post-build-variable-substitution).

- Real values live only in `clusters/<cluster>/cluster-vars.sops.yaml`, a
  SOPS-encrypted `Secret` named `cluster-vars` (namespace `flux-system`).
- Each child Flux `Kustomization` consumes them via
  `spec.postBuild.substituteFrom: [{kind: Secret, name: cluster-vars}]`.

Inspect or edit a cluster's variables:

```sh
SOPS_AGE_KEY_FILE=age.agekey sops -d clusters/the-intersect/cluster-vars.sops.yaml
SOPS_AGE_KEY_FILE=age.agekey sops clusters/the-intersect/cluster-vars.sops.yaml   # edit in place
```

> **Note:** Flux substitutes *every* `${...}` in a Kustomization's output. Any
> literal shell variable in a ConfigMap/script (e.g. `${HOME}`) must be escaped
> as `$${HOME}` so Flux passes it through unchanged.

## Adding a new application

1. Create the overlay:
   ```
   apps/<app>/the-intersect/
   ├── kustomization.yaml
   ├── <workload>.yaml
   └── secret.yaml          # encrypted with SOPS, if needed
   ```
2. Add an `apps-<app>` Flux `Kustomization` to
   `clusters/the-intersect/apps.yaml` with `dependsOn: [{name: infra-configs}]`,
   `path: apps/<app>/the-intersect`, and (if it uses `${TOKEN}` values) a
   `postBuild.substituteFrom` of the `cluster-vars` Secret. Add `spec.decryption`
   (`sops-age`) if the overlay contains SOPS-encrypted files.
3. Add any new namespace to `clusters/the-intersect/namespaces.yaml`.
4. Encrypt secrets before committing.

## Flux operations

```sh
flux get kustomizations                          # state of every Kustomization
flux reconcile source git flux-system            # force an immediate pull from git
flux reconcile kustomization apps-jellyfin --with-source   # reconcile one app + source
```

To re-bootstrap (or set up a new cluster):

```sh
flux bootstrap github \
  --owner=lfprocks \
  --repository=homelab \
  --branch=main \
  --path=clusters/<cluster-name>
```
