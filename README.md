# homelab

GitOps configuration for my homelab Kubernetes clusters, continuously
reconciled by [Flux](https://fluxcd.io/) **v2.3.0**. Flux watches this Git
repository and applies its contents to the cluster ŌĆö git is the single source of
truth for everything that runs.

## Clusters

| Cluster                                     | Status      | Notes                                                       |
| ------------------------------------------- | ----------- | ----------------------------------------------------------- |
| [`the-intersect`](clusters/the-intersect)   | **Active**  | The live cluster. Documented in detail below.               |
| `libvirt-unraid`                            | Deprecated  | Archived under [`clusters/_archive/`](clusters/_archive).   |
| `solar-pi`                                  | Deprecated  | Archived under [`clusters/_archive/`](clusters/_archive).   |

`libvirt-unraid` and `solar-pi` are no longer in use. Their entrypoints were
moved to [`clusters/_archive/`](clusters/_archive) to preserve their
configuration and history; nothing there is reconciled. See the
[archive README](clusters/_archive/README.md) for what was kept and how to
restore one.

## Repository layout

The repo follows the Flux
[monorepo structure](https://fluxcd.io/flux/guides/repository-structure/): each
cluster has an entrypoint under `clusters/`, which fans out to shared
`infrastructure/` and `apps/` directories. Common pieces live in `shared/` and
`base/` layers; each cluster opts in through its own overlay.

```
clusters/
  the-intersect/            # active cluster entrypoint (Flux Kustomizations)
  _archive/                 # deprecated clusters, not reconciled
infrastructure/
  controllers/
    shared/                 # controllers shared across clusters
    the-intersect/          # the-intersect controllers
  configs/
    the-intersect/          # the-intersect cluster config (issuers, gateways, ...)
apps/
  <app>/
    base/                   # shared base manifests for an app (where present)
    the-intersect/          # the-intersect overlay (kustomize over ../base)
scripts/                    # decrypt.sh / encrypt.sh / validate.sh helpers
```

## The `the-intersect` cluster

`the-intersect` is the only active cluster. It uses [Cilium](https://cilium.io/)
as the CNI and [Rook-Ceph](https://rook.io/) for storage. Flux is bootstrapped
against `clusters/the-intersect`, and everything the cluster runs is described by
the manifests reachable from that directory.

### How Flux is wired

`flux bootstrap` installs the Flux controllers and a root sync that points at
`clusters/the-intersect`. The root sync applies every YAML in that directory:

```
clusters/the-intersect/
  flux-system/
    gotk-components.yaml    # Flux controllers (generated by `flux bootstrap`)
    gotk-sync.yaml          # GitRepository + root sync Kustomization (generated)
    kustomization.yaml      # patches the root sync to enable SOPS decryption
  namespaces.yaml           # application namespaces (privileged PSA + gateway access)
  cluster-vars.sops.yaml    # SOPS-encrypted Secret of cluster-wide variables
  infrastructure.yaml       # shared-infra-controllers, infra-controllers, infra-configs
  apps.yaml                 # one Flux Kustomization per application
```

#### 1. Source and root sync ŌĆö `flux-system`

`flux-system/gotk-sync.yaml` defines the `GitRepository` everything flows from.
It tracks the `main` branch and polls every **1 minute**. The same file defines
the root `Kustomization` (`flux-system`), which reconciles
`./clusters/the-intersect` every **10 minutes** with `prune: true` ŌĆö applying
`namespaces.yaml`, `cluster-vars.sops.yaml`, `infrastructure.yaml`, and
`apps.yaml`. The `gotk-*` files are generated by `flux bootstrap` ŌĆö **do not edit
them by hand**.

`flux-system/kustomization.yaml` is the one hand-maintained piece here: it
patches the root `Kustomization` to add `spec.decryption` (SOPS, via the
`sops-age` secret) so the root sync can decrypt and apply
`cluster-vars.sops.yaml`. It is applied as a patch so it survives `flux
bootstrap` regenerating `gotk-sync.yaml`.

#### 2. Infrastructure ŌĆö `infrastructure.yaml`

Three `Kustomization` objects, chained with `dependsOn`:

- **`shared-infra-controllers`** ŌåÆ `./infrastructure/controllers/shared`
  (interval 1h, `wait`, `prune`). Controllers common to every cluster.
- **`infra-controllers`** ŌåÆ `./infrastructure/controllers/the-intersect`
  (interval 1h, `wait`, `prune`). Cluster-specific controllers.
- **`infra-configs`** ŌåÆ `./infrastructure/configs/the-intersect`, which
  **`dependsOn` both controller layers** so config (issuers, gateways, storage
  classes, etc.) is applied only after the controllers that consume it are ready.

#### 3. Apps ŌĆö `apps.yaml`

Each application has **its own** `Kustomization` (`apps-<name>`) pointing at
`apps/<app>/the-intersect`. Every app **`dependsOn infra-configs`** (interval
10m, `wait`, `prune`, `timeout: 5m`), so workloads roll out only after their
supporting infrastructure is ready. A few declare extra dependencies ŌĆö e.g.
`apps-adsbx-live-notifier` also `dependsOn apps-pulsar-notification-pipeline`.

## Secret & variable management

### SOPS-encrypted secrets

Secrets are committed encrypted with [SOPS](https://github.com/getsops/sops) +
[age](https://github.com/FiloSottile/age). `.sops.yaml` encrypts only the
`data`/`stringData` fields of `*.sops.yaml` files (`encrypted_regex:
"^(data|stringData)$"`), and Flux decrypts them in-cluster with the `sops-age`
secret.

```sh
# decrypt to stdout / edit in place
SOPS_AGE_KEY_FILE=age.agekey sops -d apps/<app>/the-intersect/secret.sops.yaml
SOPS_AGE_KEY_FILE=age.agekey sops apps/<app>/the-intersect/secret.sops.yaml

# push the age key into the cluster so Flux can decrypt
make age-secret-to-cluster
```

Helper scripts live in `scripts/` (`decrypt.sh`, `encrypt.sh`, `validate.sh`).

### Variable substitution (`cluster-vars`)

Values that are not secret credentials but should not be published in cleartext ŌĆö
hostnames, domains, LAN IPs, LoadBalancer ranges, the ACME email ŌĆö are kept out
of the manifests as `${TOKEN}` placeholders and resolved at reconcile time via
Flux [post-build variable substitution](https://fluxcd.io/flux/components/kustomize/kustomizations/#post-build-variable-substitution).

- Real values live **only** in `clusters/<cluster>/cluster-vars.sops.yaml`, a
  SOPS-encrypted `Secret` named `cluster-vars` (namespace `flux-system`).
- Each child Flux `Kustomization` consumes them via
  `spec.postBuild.substituteFrom: [{kind: Secret, name: cluster-vars}]`.

```sh
SOPS_AGE_KEY_FILE=age.agekey sops -d clusters/the-intersect/cluster-vars.sops.yaml
```

> Flux substitutes *every* `${...}` in a Kustomization's output. Any literal
> shell variable in a ConfigMap/script (e.g. `${HOME}`) must be escaped as
> `$${HOME}` so Flux passes it through unchanged.

## Endpoint naming & obfuscation methodology

This is a **public** repository, so it must not reveal the cluster's domains or
its serviceŌåÆendpoint map. Tokenization alone isn't enough: a token like
`${DOMAIN_LFPWEATHER_COM}` re-encodes the very value it's meant to hide, and a
literal subdomain such as `jellyfin.${...}` still maps a service to its URL.

To close that, **domains and subdomains are referenced by opaque, two-word
codenames** ([NICKA](https://en.wikipedia.org/wiki/Code_name#United_States)-style,
e.g. *Compass Call*, *Rivet Joint*). The codenames carry no meaning; the
codenameŌåÆreal-value legend exists **only** inside the encrypted `cluster-vars`
Secret.

### The scheme

| Concern        | Token form                  | Illustrative placeholder   |
| -------------- | --------------------------- | -------------------------- |
| Domain         | `${DOMAIN_<WORD_WORD>}`     | `${DOMAIN_QUIET_HARBOR}`   |
| Subdomain      | `${SUB_<WORD_WORD>}`        | `${SUB_SILVER_RANGER}`     |

(The codenames above are made-up placeholders for illustration ŌĆö the real
tokens, and what they map to, are only in the encrypted Secret.)

An ingress hostname is composed from a subdomain token and a domain token:

```yaml
# apps/<app>/the-intersect/httproute.yaml
hostnames: ["${SUB_SILVER_RANGER}.${DOMAIN_QUIET_HARBOR}"]
```

The same codenames are used everywhere a domain would otherwise leak ŌĆö embedded
URLs, `cookie-domain`/`whitelist-domain` flags, ACME/admin emails, and in
`infrastructure/configs/the-intersect/gateways.yaml` the Gateway **listener
names**, **TLS certificate secret names**, and the Cilium LB-IPAM
**sharing-keys** are all codename-derived (e.g. `common-star-tls`,
`ext-shared-01`) rather than spelling out a domain.

> **Not obfuscated (by design):** the internal Kubernetes label/annotation prefix
> `k8s.lfp.rocks/*` (e.g. `k8s.lfp.rocks/service`, `.../gpu`) is left as-is. It is
> an internal label namespace tied to physical node labels and a Kyverno policy,
> never resolved externally, so renaming it would carry operational risk for no
> confidentiality gain.

### Resolving a codename

```sh
# show the full codename -> real-value legend (requires age.agekey)
SOPS_AGE_KEY_FILE=age.agekey sops -d clusters/the-intersect/cluster-vars.sops.yaml
```

### Adding a new endpoint

1. Pick an **unused** two-word codename for the subdomain (and the domain, if new).
2. Add the mapping to the encrypted Secret:
   ```sh
   SOPS_AGE_KEY_FILE=age.agekey sops clusters/the-intersect/cluster-vars.sops.yaml
   #   stringData:
   #     SUB_<NEW_WORD_WORD>: <real-subdomain-label>
   ```
3. Reference the tokens in the manifest as `${SUB_X}.${DOMAIN_Y}`, and ensure the
   app's Flux `Kustomization` includes the `cluster-vars` `substituteFrom`.
4. Never write the real subdomain/domain (or a descriptive token name) into a
   tracked file ŌĆö the legend stays in `cluster-vars` only.

## Common operations

```sh
flux get kustomizations                                   # state of every Kustomization
flux reconcile source git flux-system                     # force an immediate pull from git
flux reconcile kustomization apps-jellyfin --with-source  # reconcile one app + its source
```

### Bootstrapping a cluster

```sh
flux bootstrap github \
  --owner=lfprocks \
  --repository=homelab \
  --branch=main \
  --path=clusters/<cluster-name>
```

## Prerequisites

- [`flux`](https://fluxcd.io/flux/installation/) CLI and `kubectl` with cluster access
- [`sops`](https://github.com/getsops/sops) + [`age`](https://github.com/FiloSottile/age)
- The repo's age private key at `age.agekey` (git-ignored ŌĆö never committed)
